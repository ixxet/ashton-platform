# Milestone 2.0 Reconciliation — 2026-04-10

## Purpose

Record the narrow honest Milestone 2.0 ruling after the cross-repo hardening
pass.

## Findings

- APOLLO needed real boundary hardening, not a new feature line: graceful
  shutdown, request bounds, bounded NATS handling, single shared-parser event
  truth, and workout safety.
- ATHENA did not need a storage leap, but it did need service-stop and publish
  resilience hardening.
- the gateway did not need a broader control plane, but it did need stricter
  caller, manifest, and request validation.
- the live ATHENA deploy does **not** move in this milestone: the truthful
  deployed line remains the bounded `v0.4.1` edge path with quick-tunnel
  ingress and no proved history path.
- proposal/apply safety remains explicit by staying narrow: gateway is still
  read-only, APOLLO trusted-surface writes stay capability-gated, and no new
  hidden write orchestration landed.

## Audit Claim Ledger

### True

- APOLLO runtime boundaries needed hardening
- APOLLO workout read/write safety needed hardening
- ATHENA service shutdown and publish resilience needed hardening
- gateway caller/manifest/request boundaries needed hardening
- deploy truth needed an explicit unchanged ruling

### False

- APOLLO still needed a parallel anonymous identified-arrival parser
- ATHENA had to widen to persistent file-backed live deployment now
- gateway audit persistence policy was missing or undefined
- Milestone 2.0 needed frontend/demo widening

### Partial

- CLI/internal coherence is acceptable where both surfaces share domain
  services, but the split still needed explicit documentation
- proposal/apply safety is acceptable because the current write boundaries stay
  narrow and trusted-surface-gated, but the docs needed to say that plainly

### Stale

- ATHENA history planning text that still implied durable observation
  persistence had not started
- APOLLO docs that still described the temporary anonymous pre-parse exception
  as the active ingest boundary

## Must Fix Now

- APOLLO: graceful shutdown, HTTP timeouts, bounded body decoding, bounded NATS
  handler contexts/drain, single shared-parser lifecycle path, workout list
  batching, bounded exercise-count writes
- ATHENA: graceful shutdown, HTTP timeouts, bounded publish retry/backoff,
  bounded dedupe memory
- gateway: constant-time caller-secret comparison, declared-argument
  enforcement, manifest symlink/path hardening, bounded tool-call body decode
- docs: make the repo/runtime versus deployed-truth split explicit

## Must Prove Now

- APOLLO duplicate arrival/departure replay, malformed/oversized body failure,
  and shutdown behavior
- ATHENA transient publish retry, bounded dedupe memory, history replay, and
  shutdown behavior
- gateway bad caller token, undeclared/wrong-type arguments, audit fail-closed,
  and manifest escape rejection
- deployed ATHENA truth remains the existing bounded live line

## Defer

- ATHENA Postgres-backed ingress storage
- ATHENA durable-history live rollout and PVC wiring
- gateway routed writes, approvals, rate limiting, and deployment proof
- meaningful frontend/demo/product widening
